## Why This Shape Of Work Fits Me

On a lot of teams I've been the first frontend hire, which usually means more than writing UI. It means choosing the stack, setting standards, building the component layer, shaping the delivery pipeline, and then making sure new engineers can actually ramp into that system without friction.

That work pattern carried through product roles in threat intelligence, iGaming, crypto, and security platforms, and it is a big part of why I gravitated toward AI product engineering. The most interesting problems still look the same to me: ambiguous brief, messy inputs, high-leverage interface, and a lot of room to turn raw capability into a polished tool.
